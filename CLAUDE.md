## Project Overview

Vaadin 25.1 + Spring Boot 4.0 web application using the Vaadin Flow (server-side Java) development model with the Aura theme. Java 21.

## Build & Run Commands

```bash
./mvnw                            # Run in dev mode (default goal: spring-boot:run)
./mvnw clean package              # Production build (JAR in target/)
./mvnw test                       # Run all tests
./mvnw test -Dtest=ClassName      # Run a single test class
```

The app runs on port 8080 (configurable via `PORT` env var).

## Architecture

**Vaadin Flow model**: UI is built entirely in server-side Java. Each view is a Java class extending a Vaadin layout component, annotated with `@Route`. The framework syncs UI state to web components in the browser automatically.

- `Application.java` — Spring Boot entry point, implements `AppShellConfigurator` for theme/stylesheet config
- Views go in subpackages of `com.example.specdriven` (e.g., `exampleview/ExampleView.java`)
- Routes are declared via `@Route("path")` and page titles via `@PageTitle`
- Custom CSS goes in `src/main/resources/META-INF/resources/styles.css`
- `src/main/frontend/generated/` is auto-generated by Vaadin — do not edit

## Key Configuration

- `application.properties` — server port, Vaadin settings, component scanning packages
- `vaadin.allowed-packages` must include any new package that contains Vaadin components or Spring beans
- `.mcp.json` — configures Vaadin docs MCP server and Playwright MCP for browser automation

## MCP Tools

This project has MCP servers configured for:
- **Vaadin** (`mcp.vaadin.com/docs`) — use for Vaadin component docs, APIs, and best practices
- **Playwright** — use for browser-based testing and automation
